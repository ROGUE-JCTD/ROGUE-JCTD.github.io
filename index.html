About ROGUE

The Rapid Open Geospatial User-Driven Enterprise (ROGUE) Joint Capability Technology Demonstration (JCTD) is committed to providing an innovative approach to geospatial data development, management, and sharing that leverages the combined strengths of institutional interactions, provides provenance, and enables rich collaboration. These objectives are being accomplished by leveraging a suite of open source software, based on open standards, in order to allow the widest availability and interoperability. This approach helps to overcome current challenges that organizations experience when trying to collaboratively develop and share geospatial information.

Principles Behind ROGUE

ROGUE is based on the principle that there is greater power in opening up geospatial information for contribution by other teams and organizations. Git and GitHub are proof that the model works for community development and OpenStreetMap is proof that it works for maps. GeoGit allows us to open up data and let distributed groups contribute and collaborate.

  • Rapid: Sharing information as soon as possible
  • Open: Open source software and open standards
  • Geospatial: Focused on Location and Time
  • User-driven: Users can edit and contribute
  • Enterprise: The OpenGeo Suite provides a powerful spatial data infrastructure

Leveraging Open Source Technology

ROGUE leverages open source software and is built using the OpenGeo Suite. In addition, ROGUE has been a driver in the development of GeoGit - which provides distributed versioned editing for geographic data. These are the core software projects that form the foundation for ROGUE:

  • GeoGit
  • GeoNode
  • OpenGeo Suite
  • OpenLayers 3

In order to take full advantage of GeoGit, the ROGUE team has developed Arbiter for mobile editing and MapLoom for web-based editing. Both of these projects are open source as well. We've also augmented GeoNode so that it works with GeoGit.

  • Arbiter
  • MapLoom
  • ROGUE GeoNode

For Developers

Although all of the components used for the ROGUE-JCTD are open source and available for free on the internet, downloading, installing and configuring all of the components is a significant undertaking. We recommend using Chef, Vagrant and VirtualBox to automatically install and configure the ROGUE components. Follow the steps below to stand up your own ROGUE instance.

Note: Make sure you are using Ruby 1.9.3+ before starting.

  1. Install VirtualBox
  2. Install Vagrant 1.3.5+
$ gem install berkshelf
$ vagrant plugin install vagrant-berkshelf
$ git clone https://github.com/ROGUE-JCTD/rogue-chef-repo.git
$ cd rogue-chef-repo
$ nano Vagrantfile  # modify the Vagrantfile based on your network configuration.
$ berks install
$ vagrant up
